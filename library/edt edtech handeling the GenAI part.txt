For AI, we’re keeping it as a separate RAG service in Python FastAPI. The main backend will send student doubt requests to that service, where we’ll fetch relevant chapter-level content from the vector database, build the right context, and return a grounded answer. This helps us make the AI more accurate and exam-focused instead of generic. We’ll also use caching and answer reuse for repeated doubts so the system stays faster and more efficient. Overall, the plan is to make AI a strong support layer inside the product, not the whole product itself.
