We’ll run this in tight weekly sprints and keep the scope disciplined around MVP. Week 0 is just setup—repo, base backend, Postgres and Redis wired, auth scaffold, and a deployable skeleton. Sprint 1 is core models and auth—users, courses, chapters, basic APIs and UI shell. Sprint 2 we lock content—MCQs, key points, chapter APIs, and seed data so the system actually has substance. Sprint 3 is the MCQ engine—attempts, wrong question tracking, retry flow, and caching hot reads. Sprint 4 is the critical piece—the RAG-based doubt solver with answer reuse; we don’t hit the LLM blindly, we check similarity first to control cost and latency. Sprint 5 we layer progress tracking and a usable dashboard—stats, completion, basic insights. Sprint 6 is test system—chapter tests, submissions, scoring. Sprint 7 we bring monetization—plans, usage limits, paywall triggers, enforced via Redis for speed. Sprint 8 is polish—saved doubts, daily tasks, tone control in AI responses so it doesn’t sound generic. Sprint 9 is stability—queues for heavy ops, rate limiting, logging, deploy behind Nginx. Last sprint is buffer—fixes and performance tuning. No overengineering—Postgres as source of truth, Redis aggressively for cache and limits, queue for async jobs, and we scale horizontally when needed, not before. The goal is a working, paid MVP in ~6–8 weeks, not a perfect system.
